## P2 — Shareable URL encoding

**What:** Encode the current function expression (and any relevant settings) into the URL hash so anyone can share a wave by copying the URL.

**Why:** This is the viral mechanic that turns a local toy into something people share — "surf this equation" as a link.

**Pros:** Zero backend required. ~20 lines of code. URL is human-readable (`#f=sin(x*y)*cos(t)`).

**Cons:** URL can get long for complex nested expressions; may need to add optional URL shortening later.

**Context:** On page load, parse the hash and pre-fill the expression editor. On expression change, update the hash. Use `window.location.hash` directly. Do this after the expression editor is stable to avoid encoding format churn.

**Effort:** S (human: ~2 hrs) / S (CC: ~5 min)

**Priority:** P2

**Depends on:** Expression editor (Expansion #1 — accepted in scope)

## P3 — LocalStorage high scores per function

**What:** Store the player's best score for each function configuration in `localStorage`. Show "PB: 1240" during a run when the player's current score is approaching their record.

**Why:** Each preset in the function gallery becomes a personal challenge with memory. Gives players a reason to replay the same wave.

**Pros:** No backend. Pure browser. Works offline.

**Cons:** Resets if user clears storage; can't persist across devices.

**Context:** Key = stable hash of the expression string. Value = `{ score: number, date: ISO string }`. Show personal best in the score HUD. On wipeout, flash "New PB!" if beaten.

**Effort:** S (human: ~2 hrs) / S (CC: ~5 min)

**Priority:** P3

**Depends on:** Function gallery (Expansion #3 — accepted in scope)

## P3 — Screenshot / poster export

**What:** A keyboard shortcut (e.g. `P` for poster) that downloads the current canvas frame as a high-res PNG with the equation captioned at the bottom in the retro monospace font.

**Why:** The image is beautiful standalone — a stunning retrowave landscape with a math equation. Shareable on its own even without the game context.

**Pros:** `canvas.toDataURL()` + anchor download = ~20 lines. The equation overlay (already in scope) makes the captioning trivial.

**Cons:** Minimal. The only risk is the download interrupting the game loop (pause during export).

**Context:** Pause the game loop during export. Draw the equation caption below (or overlaid on) the current frame on a fresh offscreen canvas at 2x resolution, then trigger download.

**Effort:** S (human: ~2 hrs) / S (CC: ~5 min)

**Priority:** P3

**Depends on:** Equation display overlay (Expansion #4 — accepted in scope)
