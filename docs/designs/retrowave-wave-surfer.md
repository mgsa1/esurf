## Vision

### 10x Check
A social math playground: you type `sin(x*y)*cos(r*t)` and a friend opens your URL and surfs your exact wave on a stunning retrowave grid. When they nail a ridge grind, they screenshot it and the equation is embedded in the image caption. Every function is a universe — saddle points create half-pipes, ripple functions create mogul fields. The math *is* the game design.

### Platonic Ideal
You open it. No tutorial. Purple-to-black sky, neon grid receding to a city silhouette horizon. A wave is already coming. Press space — you're riding. You look down and see `z = sin(x)·cos(y·t)`. Tap a slider and the terrain reshapes under you mid-ride. Wipe out on a ridge and tumble beautifully. Then edit the function and send it to a friend. It's magic. It's math. The two things are the same.

## Architecture

### View model
- **Camera:** Fixed on beach, facing the ocean (looking at the X/Y plane)
- **Wave:** Height = `z = f(x, y, t)` projected onto Y axis — rendered as 2D wave profile
- **Player:** 2D surfer sprite controlled with WASD, Dolphin-Olympics-style physics
- **Loop:** Endless runner — wave scrolls from right, function evolves with `t`

### Expression parsing
- **Library:** `expr-eval` (sandboxed math parser, NOT `eval()`)
- Variables: `x` (horizontal position), `y` (depth/phase), `t` (time in seconds)
- SECURITY: expr-eval has no access to `window`, `document`, or any JS runtime. Do NOT replace with `eval()`.

### Module structure
```
src/
  constants.js        ← GRAVITY, WIPEOUT_THRESHOLD, WORLD_SCALE, AMPLITUDE, etc.
  WaveEngine.js       ← expr-eval wrapper, formula management, height array
  Player.js           ← state machine, Dolphin Olympics physics
  Renderer.js         ← all Canvas 2D drawing
  ParticleSystem.js   ← wipeout tumble + particle burst
  InputHandler.js     ← WASD keyboard state
  FunctionEditor.js   ← DOM: text input + clickable function blocks
  FunctionGallery.js  ← DOM: preset wave selector
  ScoreSystem.js      ← speed, air time, combo tracking
  main.js             ← GameLoop, wires everything together
```

### Player state machine
```
IDLE → SURFING ←→ AIRBORNE
                ↓         ↓
           WIPING_OUT ←──┘ (bad landing)
                ↓
           SURFING (reset)
```

## Accepted Scope (V1)

- **Typed expression editor** with free-text input AND clickable function building blocks
  - Hybrid: click blocks to compose, OR type freely — both feed the same expr-eval parser
- **Function gallery:** 6-8 named preset waves (Tsunami, Saddle Point, Ripple Tank, Mogul Field, etc.)
- **Equation display overlay:** `z = ...` rendered in retro monospace font on canvas
- **Wipe-out animation:** surfer tumble + particle burst on crash (0.5s, ~15 particles)
- **Speed streak lines:** motion-blur dashes behind surfer proportional to speed

## Critical Implementation Guards

1. `dt = Math.min(dt, 0.1)` in game loop — prevents wave jump on tab resume
2. `isFinite(waveHeight) ? waveHeight : 0` in WaveEngine — prevents NaN in physics
3. `if (!ctx) { showError("Canvas unavailable"); return; }` in Renderer init
4. `// SECURITY: expr-eval is sandboxed — do NOT replace with eval()` in WaveEngine
5. Player state guard: ignore WIPEOUT triggers while already WIPING_OUT
